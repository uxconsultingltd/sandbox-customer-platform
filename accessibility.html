Customer Portal is operated by Scottish Enterprise and Partners.

Scottish Enterprise and the delivery partners are committed to making our services accessible to everyone who could benefit from them, irrespective of any disability they may have.

Scottish Enterprise is committed to making this website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018. This website is partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard, due to the issues listed on this page.

This statement was prepared on 29/09/2022. It was last reviewed on 29/09/2022.

How we approach accessibility

We regularly test our websites with disabled users to identify and address any issues. We have a testing team, who test new features and changes to the site for accessibility issues. This website was last tested in 30/09/2022. The test was carried out by our internal test team.

As well as aiming to build our website in a way that’s easy for all users to use and navigate, we’re also working toward making the language we use as simple as possible to understand by using plain English principles.

AbilityNet has advice on making your device easier to use if you have a disability.

Accessibility problems we've found

During testing of our website, we’ve found the following problems, which we aim to fix by MM/YYYY. Once we’ve fixed a problem, we’ll remove it from this list.

Group 1 - Expected fixed by November 2022:

  • HTML Language - Wrong en-us should be en-gb.
  • Font Sizing - Font sizes in the entire site is set to pixels and will prevent user scaling font sizes. Change to em/rem.
  • Footer - Footer links should be wrapped in a nav tag and UL, LI's.
    • Need optimised to be presented as a proper navigation landmark.
    • Add aria label describing the nav tags purpose.
  • Header Banner Area - The banner area has a role navigation set on it instead of role="banner".
  • Header Skip Link - Skip to main content :hover state doesn't pass colour contrast guidelines. Toggle background. Keep style of the primary button.
  • Header – Hamburger Nav - Same text is already set in buttons inner sr-only span tag. Remove the title tag from button.
  • Header – Hamburger Nav - Triplicated text : title - Inner html - aria-label.
    • Keep anchor text, remove title and aria label.
  • • Header – Hamburger Nav - 2nd list item "sign in" has role="none" which could confuse some Assistive Techs when announcing how many items are in the list.
    • Remove Title and aria-label when logged in.
    • Remove role=”none”

Terms and conditions page

  • Add aria-required to term acceptance checkbox. (Investigation Required)
    • Add aria required or sr-only span, explaining the fields required.
  • Checkbox hover state should be consistent with other inputs, and sizing.

Azure B2C

  • HTML Lang = en-us
    • Should be en-gb (to help with any localisation).

Group 2 - Expected fixed by December 2022:

  • Header Logo - Use of title, aria-label and inner HTML text for link, essentially repeats the same text up to 3 times on some Assistive Techs.
    • Set alt text to empty string.

Group 3 – Expected fixed by January 2023:

  • Offline Banner - Offline banner has a div and event handler behaving as a button, to close the notification, however this is not keyboard accessible, use native HTML.
    • Also the div wouldn't have fed back it's purpose as it had no inner html text to explain itself. (Investigation required)
    • Font Sizing - Message text is 12px, which falls below min recommended accessible size (16px). (Investigation required)
  • Footer - Copyright text should be a CSS background image, not text.
    • Change to CSS background img or put copyright logo inside a span with an aria hidden = true
  • Invite email – Encoding markup - We=92ve provided a link for you to create your account. (Investigation required)
